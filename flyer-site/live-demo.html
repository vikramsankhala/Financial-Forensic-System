How the live demo works

This static page documents the live experience. The actual live demo is powered by a local Express API + SQLite dataset and a React UI.

  • Seeded alert, case, and transaction data stored in SQLite.
  • Server pushes new alerts every ~8 seconds for realism.
  • Metrics update via streaming events (SSE).
  • UI renders in under 100ms to mimic real-time operations.
Investor + operator talking points
  • Sub-100ms fraud scoring with explainable signals.
  • Auto-case creation for critical alerts.
  • Unified view across channels and roles.
  • Audit-ready trail for compliance review.
  • Rapid time-to-value with live alert streams.

Breakdown of each UI area

1. Metrics Header

Summarizes operational health: transaction throughput, alert volume, open case load, response time, and false-positive rates.

2. Alert Stream

Real-time feed with risk-scored alerts used to prioritize investigator workflows and immediate actioning.

3. Case Queue

Shows auto-generated cases with owners, priorities, and summaries so teams can triage quickly.

4. Transaction Feed

Provides context on recent activity across channels to validate alerts and surface anomalies.

5. Investigator Actions

Escalation, hold, and notification actions are documented in the case record with a full audit trail.

6. Compliance View

BSA/AML reporting hooks, CTR/SAR readiness, and entity network context are embedded into the workflow.
